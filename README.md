## 4. 목표 서비스 구현 및 실제 구현 정도

1. 목표 설정
   - 영화 평점, 선호&비선호하는 장르, 좋아하는 영화를 기반으로 한 영화추천 서비스 구성
   - 커뮤니티 서비스 구성
   - HTML, CSS, JavaScript, Django, REST API, DataBase 등을 활용한 실제 서비스 설계
   - 사이트 배포

2. Design Thinking

   - 페르소나1 (타겟)
     - 영화 커뮤니티에서 활동하고 싶은 20대 학생 A씨  
     - 공포, 전쟁 영화를 싫어해서 절대 관련 포스터도 보기 싫어함
     - 좋아하는 영화 장르가 뚜렷하고 좋아하는 영화와 비슷한 영화들을 추천받고 싶음
   - 페르소나2
     - 평소 평점 위주로만 추천하는 영화 사이트들에서 싫어하는 장르여도 높은 평점인 영화를 추천 받는게 불만인 직장인 B씨
     - 직접 평점을 매기고 영화를 추천받고 싶어함

## 7. 필수 기능

### Movies Homepage

- 취향에 따른 영화 추천

  - "내 프로필"의 "My FavMov" 에 등록된 선호하는 장르, 비선호하는 장르, 좋아하는 영화 그리고 사이트 내 매겨진 평점을 기반으로 영화추천 서비스 제공 
  - 추천 알고리즘은 좋아하는 영화의 장드들과 선호하는 장르를 합친뒤 비선호 하는 장르를 제거 한 후, 사이트 내 매겨진 평점을 기반으로 추천을 하게 됨
  - 예를 들어, 공포 영화 장르를 싫어하는 사람은 포스터도 보기 싫어하기 때문에 제거에 우선순위를 두었고, 좋아하는 영화는 추천하는 영화에서 제외함
  - 평점을 의도적으로 높이거나 낮출 수 있기 때문에 평점 댓글은 1개만 작성하도록 함

  



### Signup & Login 

- 비로그인 상태로 Community에 좋아요를 눌렀을 시 Login page로 연결

- 비로그인 상태로 Movies Detail page에서 [댓글을 작성하려면 로그인하세요.] 클릭시 Login page로 연결

  



### Movies Detail

- 댓글 기능에 rank 평점 기능 추가
- 평점이 없을 경우는 DB 평점이 노출
- 평점이 1개 이상 등록될 시 사이트 내 유저들이 등록한 평균 평점이 노출
- 영화 좋아요 버튼을 비동기 방식으로 구현하여서, profile 취향 목록에 추가, 제거가 되게 구현함





 ### 내 프로필

- 취향 변경을 통해 My FavMov(선호하는 장르, 비선호하는 장르, 좋아하는 영화) 다중 선택 가능
- My FavMov에 입력된 정보를 통해 Movies Homepage에 추천영화 서비스 제공
- 영화 detail 페이지에서 좋아요 버튼을 누른 영화를 한눈에 확인이 가능
- 작성한 모든 게시글과, 평점 댓글을 확인이 가능





### Community

- 좋아요, 댓글 CRD
- Detail page에서 게시글 CRUD





### New Review

## 8. 느낀 점

### 최수한

 프로젝트를 시작할때 어떻게 해야할지 막막하고, 기획 시작 단계에서부터 ERD 작성, 요구사항 파악, UI/UX 화면 설계 등 지금까지 관통 프로젝트와는 달리 실전 프로젝트의 느낌이었습니다. 시간도 많다고 생각했지만, 시간이 갈수록 부족하다고 느꼈고, 프로젝트를 막힘 없이 기능 구현을 해 나갈줄 알았지만, 예외 처리, 에러 처리, 배포에서의 문제, 생각지 못한 db 오류 등 많은 문제가 있었습니다. 수업 했던 부분을 참고하고, 모르는 부분을 다시 공부하면서 전 관통 프로젝트 때 했던 경험을 살려서 개발을 하다보니 실력이 늘어 가는것을 느꼈습니다.

 프로젝트에서 팀장을 맡아서 페어인 지애님과 파트를 나눠서 공부를 해 나갔고, 프로젝트가 마무리 되는 상황에서 생각해보면 혼자 했으면 어떻게 해 나갔을지 모르겠습니다. 프로젝트를 하면서 시간이 널널 할줄 알았지만, 저녁마다 기능 구현 및 공부를 해 나가며 성취감을 느꼈고, 문제 해결을 하면서 물론 힘들었지만 재미도 느꼈습니다. 교수님에게 방향을 묻고, 프로젝트 방향을 많은 상황속에서 원활한 소통과 회의로 바꾸고, 다른 팀 분들에게도 서로 피드백을 받으며 알려주면서 배우고, 새롭게 자극 받기도 하였습니다. 

 짧을 수도 있는 이 프로젝트 기간에 좋은 팀원과 같이 프로젝트를 진행하면서 소통 및 협업 능력과 문제해결 능력, 그리고 기능 구현과 같은 개발적인 측면에서의 성장이 스스로 눈에 띄며, 자극을 받은 좋은 기회였습니다.



### 하지애

처음 시작하고 계획을 세울 때만 해도 요구사항에 있는 부분들을 마무리하고 아이디어 낸 기능들을 구현할 수 있을까 걱정이 많았습니다. 프로젝트 동안에는 최대한 처음 세웠던 일정대로 그날 계획한 부분은 그날 마무리 지으려고 노력했고 이 과정에서 팀장님과 새벽 4시까지 작업하기도 하고 팀장님은 밤을 새며 작업하기도 했습니다. 차근차근 하나씩 구현해나가다보니 프로젝트 마지막날 드디어 계획했던 모습 그대로 배포까지 마무리할 수 있었습니다. 프로젝트가 끝나가는 지금 프로젝트를 진행하는데 있어서 가장 중요한 것은 처음 설계와 소통이라 생각합니다. 저희는 프로젝트를 시작하기 전 어떻게 나아갈 지 방향성과 구현해내고 싶은 아이디어들을 위해 충분히 이야기를 나눴고 진행하면서도 계속 확인했기에 함께 큰 문제없이 프로젝트를 진행할 수 있었습니다. 

프로젝트를 하면서 어려움도 많았습니다. 오류가 나서 오류때문에 하루를 허비한 적도 있었고 각자 branch로 작업하다 merge하는 과정에서 어려움을 겪기도 했지만 그때마다 다시 공부했던 것을 돌아보고 google 검색을 통해 생각을 더 넓히기도 하고 교수님과 주위 분들에게 물어가며 해결할 수 있었습니다. 프로젝트를 통해 여러 기능들을 구현했는데 하나씩 했을 때는 이상없던 부분들이  합쳤을 때는 꼬여서 오류가 나는 것을 보고 좀 더 크게 볼 줄 알아야겠다 생각을 했습니다. 또한, 화면으로 봤을 때는 간단해 보이는 기능의 구현도 실제로 구현해내는 과정은 생각보다 간단하지 않은 경우가 많았기에 앞으로도 많이 공부해서 다음부터는 더 잘해내고 싶다 생각했습니다. 프로젝트 내내 함께 프로젝트를 진행한 수한님께 정말 감사했
